## 미션

### Part 1: 상품 이미지 업로드 (관리자 기능)
- ImageKit으로 상품 이미지를 업로드하고 DB에 URL 저장
- 상품 등록 화면에서 드래그앤드롭 또는 파일 선택으로 이미지 첨부
- 상품 목록에 업로드된 이미지가 실제로 뜨는지 확인

### Part 2: 결제 플로우
- 장바구니 → 결제 페이지 이동
- 토스페이먼츠 결제 위젯 붙이기 (테스트 키)
- 결제 승인 API는 반드시 **서버에서** 호출 (Secret Key 노출 금지)
- 주문 정보를 DB `orders` 테이블에 저장

### Part 3: 마이페이지
- 로그인한 사용자의 주문 내역 조회
- 각 주문의 상품명 / 금액 / 주문일 / 주문번호 표시
- 본인 주문만 볼 수 있도록 권한 제어

### Part 4: 배포
- Vercel에 배포하고 URL 제출
- 환경변수(토스 Secret Key, ImageKit Key)는 Vercel 대시보드에 등록

---

## 핵심 구조

```
[관리자 상품 등록] → ImageKit에 이미지 업로드 → DB에 URL 저장
[장바구니] → [토스 결제 위젯] → 서버 승인 API → DB orders 저장
[마이페이지] → 본인 user_id로 orders 조회 → 주문 내역 렌더링
```

**DB 테이블 구조 (참고):**

```
products 테이블:    id, name, price, image_url, description
orders 테이블:      id, user_id, total_price, paid_at, status
order_items 테이블: id, order_id, product_id, quantity, price
```

> 💡 **깨달아야 할 핵심:**
> 5주차까지 만든 쇼핑몰은 "가짜 쇼핑몰"이었어요. 이번 퀘스트로 **실제 돈이 흐르고(결제), 실제 파일이 저장되고(이미지), 내 주문 기록이 남는(마이페이지)** 진짜 서비스의 뼈대를 완성합니다.
> 토스페이먼츠의 Secret Key는 **절대 프론트엔드에 노출되면 안 됩니다** — 왜 승인 API를 서버에서 호출해야 하는지, 이 퀘스트를 하면서 자연스럽게 체득하게 됩니다.
