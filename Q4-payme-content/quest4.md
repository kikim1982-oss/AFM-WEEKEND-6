## 미션

### Part 1: 콘텐츠 주제 선택
다음 중 하나 선택 (또는 자유 주제):
- **유료 블로그 글**: "이 글은 500원" 패턴
- **유료 뉴스레터**: "이번 주 호는 구독자만"
- **유료 영상 강의**: "이 영상은 결제 후 시청"
- **유료 레시피/템플릿/자료 다운로드**

### Part 2: 잠금 콘텐츠 만들기
- **목록 화면**: 콘텐츠 제목 + 미리보기 일부(앞 3줄) + 가격 표시
- **잠금 화면**: 결제 전엔 전체 본문이 블러/잠금. "결제하고 열람" 버튼
- **결제 플로우**: 토스 결제 위젯 → 서버 승인 API → DB `purchases` 테이블에 기록
- **열람**: `purchases`에 `user_id + content_id`가 있으면 본문 공개

### Part 3: 구매 이력 페이지
- 로그인한 사용자의 구매한 콘텐츠 목록
- 언제든 다시 열람 가능 (재결제 X)

### Part 4: 배포
- Vercel에 배포하고 URL 제출

## 핵심 구조

```
[콘텐츠 목록 (미리보기)] → [잠금 화면] → [토스 결제] → DB purchases 저장 → [본문 열람]
                                                                 ↓
                                                       [구매 이력 페이지]
```

**DB 테이블 구조 (참고):**
```
contents 테이블:   id, title, preview, body, price
purchases 테이블:  id, user_id, content_id, paid_at, amount
```

**열람 권한 판단:**
```
purchases에 (현재 user_id + 보려는 content_id)가 있으면? → body 반환
없으면? → 결제 유도 화면 반환
```

---

💡 **깨달아야 할 핵심:**
쇼핑몰은 "물건을 산다"지만, 이 앱은 "**접근 권한을 산다**"입니다. 현대 SaaS / 콘텐츠 서비스의 핵심 패턴 — 구독 서비스, 강의 플랫폼(인프런/유데미), 유료 뉴스레터(서브스택), Medium 유료 글이 모두 같은 구조입니다.
`user_id + content_id`만 매칭되면 열람 허용, 없으면 결제 유도 — 이 작은 아이디어 하나로 인터넷의 수많은 비즈니스 모델이 돌아갑니다.

- 이 구조를 이해하면 Medium / Substack / Inflearn의 핵심 로직을 이해한 것
- 여유가 되면 "월 구독(전체 열람)" 모델도 도전 — `subscriptions` 테이블에 expires_at 추가
